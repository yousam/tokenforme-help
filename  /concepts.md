# 核心概念

本章节用尽量直观的方式，介绍 tokenfor.me 中与「普通用户」密切相关的几个核心概念，帮助你更好地理解控制台中的配置项和统计信息。

## 多模型 API 网关（Multi-LLM API Gateway）

tokenfor.me 的核心定位是 **High-quality Multi-LLM API Gateway**：

- 通过一个统一域名（参考「概览」章节中的「API 地址（Base URL）」小节）接入多家大模型服务；
- 统一鉴权、统一计费和用量统计；
- 屏蔽不同厂商之间的细节差异，减少重复配置和适配成本。

> 注意：统一域名不代表同一个 Key 可以同时访问多家模型。请见下文「Key 与分组」。

## 账号与组织

- **账号**：你在 tokenfor.me 上注册的个人账号，用于登录控制台、管理 Key 和查看账单。
- **组织 / 团队**（如果有）：
  - 用于多人协作、统一管理多个成员的 Key、用量和账单；
  - 企业/集团客户通常会使用组织级别的功能。

## API Key、AI 厂商与分组（重要）

- 在 tokenfor.me 中，**一个 Key 只对应一个 AI 厂商下的一个分组**（例如：OpenAI → GPT Standard）。
- 如果你需要同时使用 Anthropic、Gemini 等其他 AI 厂商，请**分别创建新的 Key**；
- 同一家 AI 厂商可能存在多个「分组」，不同分组代表不同的渠道体验与倍率（见下文）。


## 使用情况与账单

- 每一次通过「API 地址」发起的调用都会被记录（具体地址参考「概览」章节中的「API 地址（Base URL）」小节）；
- 控制台提供用量统计与账单视图，用于查看：
  - 调用次数 / tokens / 费用估算；
  - 不同 Key 的用量分布；
- 可在「使用情况」页面导出 CSV 明细；
- 输入/输出/缓存 token 数据均由模型供应商接口返回；
- 各模型 token 价格与官方保持一致（以官方为准）。

## 个人用户 vs 集团 / 企业用户

- **个人用户**：关注稳定访问、如何充值、费用可控；
- **集团 / 企业用户**：在以上基础上，关注团队管理、成本分摊、SLA 与售后支持。

后续章节会从这两类用户的实际使用场景出发，结合控制台界面和配置示例，帮助你一步一步用好 tokenfor.me。
