# 创建与管理 API

本章节详细介绍如何在 tokenfor.me 控制台中创建、查看和管理 API Key，以及如何为 Key 配置模型路由。

## 创建新的 API Key

1. 登录 `https://tokenfor.me` 控制台。
2. 在左侧菜单中进入「API 管理 / Keys」模块。
3. 点击「创建 API Key」或类似按钮。
4. 填写以下信息（以实际界面为准）：
   - **名称**：用于区分用途，例如 `personal-dev`, `team-project-A`；
   - **备注**（可选）：补充使用场景说明，例如「个人测试用」「后端服务调用」等；
5. 提交后，系统会为你生成一个新的 API Key。

> 提示：
> - 创建成功后，请立即复制并妥善保存该 Key；
> - 出于安全考虑，系统可能只在创建时完整展示一次。

## 配置模型路由

为 API Key 配置模型路由，可以决定该 Key 能访问哪些模型以及如何转发请求。

1. 在 API Key 列表中，找到刚刚创建的 Key，点击「编辑」或「配置」。
2. 在“模型路由”或类似模块中，选择你希望启用的模型供应商，例如：
   - OpenAI
   - Anthropic
   - Gemini
   - 其他已接入的模型供应商
3. 对于每个供应商，你可以：
   - 启用 / 禁用；
   - 选择具体可用的模型列表（如 `gpt-4`、`claude-3-opus`、`gemini-1.5-pro` 等，以实际支持为准）；
   - 配置相关参数（如优先级、限流策略等，如有）。
4. 保存配置后，该 Key 即可通过 `https://api.tokenfor.me` 访问相应模型。

对普通用户而言，一般只需要：

- 确认需要用到的模型供应商已勾选；
- 在客户端正确填写模型名称与 API Key。

## 查看与管理现有 Key

在「API 管理 / Keys」页面，你可以：

- 查看 Key 列表：名称、创建时间、状态（启用 / 禁用）、最近使用时间等；
- 编辑 Key：修改名称 / 备注、调整模型路由配置；
- 启用 / 禁用 Key：
  - 建议对不再使用或疑似泄露的 Key 进行禁用或删除；
- 删除 Key：
  - 删除后，该 Key 将无法再用于调用接口，请提前确认不会影响线上服务。

## 使用建议

- 为不同用途创建不同 Key：
  - 例如“本地开发调试”“生产环境后端服务”“某个成员个人实验”等；
  - 便于在用量统计和账单中区分来源。
- 定期审查 Key 列表：
  - 清理长时间未使用的 Key；
  - 对怀疑泄露的 Key 立即禁用或删除。

配置完成后，你即可在各类智能体或客户端中，通过 `https://api.tokenfor.me` + 对应的 API Key 调用所需模型。
