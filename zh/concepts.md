# 核心概念

本章节用尽量直观的方式，介绍 tokenfor.me 中与「普通用户」密切相关的几个核心概念，帮助你更好地理解控制台中的配置项和统计信息。

## 多模型 API 网关（Multi-LLM API Gateway）

tokenfor.me 的核心定位是 **High-quality Multi-LLM API Gateway**：

- 通过一个统一入口（`https://api.tokenfor.me`）调用多家大模型服务；
- 统一鉴权、统一计费和用量统计；
- 屏蔽不同厂商之间的细节差异，减少重复配置和适配成本。

对普通用户来说，你可以把它理解为：

> 把「很多不同地方的大模型」整合成「一个好用、稳定的统一入口」。

## 账号与组织

- **账号**：你在 tokenfor.me 上注册的个人账号，用于登录控制台、管理 Key 和查看账单。
- **组织 / 团队**（如果有）：
  - 用于多人协作、统一管理多个成员的 Key、用量和账单；
  - 企业/集团客户通常会使用组织级别的功能。

## API Key / Token

- 每个 API Key 对应一组访问权限和配额限制；
- 可以为不同用途创建不同的 Key，例如：
  - 个人开发测试用 Key；
  - 某个项目专用 Key；
  - 某个团队成员专用 Key；
- 在智能体或客户端中，只需要配置 Base URL 和 API Key 即可完成接入。

**重要提示：** 请妥善保管你的 API Key，不要在公共仓库、截图或不可信环境中泄露。

## 模型路由（Model Routing）

tokenfor.me 通过「模型路由」把多个大模型供应商统一在一起：

- 你可以在控制台中选择需要支持的模型路由，例如：
  - OpenAI 系列模型
  - Anthropic 系列模型
  - Gemini 系列模型
  - 以及其他支持的模型供应商
- 对普通用户而言，一般不需要关心背后复杂的路由逻辑，只需要：
  - 在控制台中勾选、启用你希望使用的模型；
  - 在客户端中选择对应模型名称即可。

## 使用情况与账单

- 每一次通过 `https://api.tokenfor.me` 发起的调用都会被记录；
- 控制台会提供用量统计与账单视图，帮助你了解：
  - 总调用次数 / tokens / 费用估算；
  - 不同模型、不同 Key 的用量分布；
- 个人和企业都可以据此进行成本控制和优化。

## 个人用户 vs 集团 / 企业用户

- **个人用户**：更多关注「能不能稳定访问模型」「怎么充值」「费用是否可控」。
- **集团 / 企业用户**：在以上需求基础上，还会关注：
  - 团队成员的统一管理；
  - 各部门用量与成本分摊；
  - 售后支持与服务等级（SLA）。

后续章节会从这两类用户的实际使用场景出发，结合控制台界面和配置示例，帮助你一步一步用好 tokenfor.me。
